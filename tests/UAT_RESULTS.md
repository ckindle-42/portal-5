## Observations for future fixes

### P1 ‚Äî Pipeline Ollama fallback timing (10 FAILs)
Tests T-02, P-D20, P-B01, P-D10, P-D01, P-D18, P-V11, P-R04, P-B05, P-R01 all fail
with empty retries even with clean memory. Root cause: pipeline health-checks MLX as
unavailable during model switch transitions (state=switching) and falls back to Ollama
models (qwen3-coder, deepseek-r1, qwen3-vl) that cannot handle the task. The MLX
model IS loading successfully, but the routing decision is made before it's ready.
Fix: extend router tolerance for switching-state backends, or add a retry-with-delay
path in router_pipe.py.

### P1 ‚Äî Large model cold-load timing (FIXED: --no-unload + model readiness wait)
WS-16 (compliance, 35B MoE) went from 3-empty-retry FAIL to 6/6 PASS (546s) when run
with --no-unload + pre-warmed model. Root cause: driver always /unload on startup,
then fires test immediately without waiting for cold-load (30-90s for 35B). Each
empty retry triggers another cold-load, producing 870s of empty responses.
Fix applied: added _wait_for_mlx_ready() check + --no-unload flag to driver.
WS-17 improved to 5/6 WARN (83%). The gate's verify block also evicted pre-warmed
models; --no-unload now skips those checks too.

### P2 ‚Äî Assertion keyword tuning (3 FAILs, ~3 WARNs)
T-01 triggers on "the output would be" as a prediction false-positive. P-R02 expects
the model to ask for requirements before outputting architecture but the model jumps
to solution. P-S04 expects literal "scope"/"authorization" keywords. Several
re-classified WARNs (A-01, WS-02, P-D04, etc.) passed ‚â•70% of assertions but missed
1-2 narrow keywords. Fix: broaden keyword lists or mark narrow assertions as
critical:False in the test catalog.

### P3 ‚Äî Known hardware limitations (1 FAIL)

RL-trained for STEM/math, not code generation. Produces empty HTML blocks on M4.
This is expected behavior, not a bug. CC-01-granite41-30b recovered and PASSED on
clean-memory re-run.

### P3 ‚Äî Portal/OWUI feature gaps (3 FAILs)
A-08 (cross-session memory): Open WebUI memory feature doesn't persist named facts
across separate chats as the driver expects. T-08 (ComfyUI image): image was
generated but Playwright didn't download the file attachment. T-11 (security MCP):

### P3 ‚Äî Model capability floor (1 FAIL)

HTML but misses structural elements. Bench exists to surface capability differences;
this is by design per P5-BENCH-001.

The gate's 5-level recovery ladder and ComfyUI/Ollama pressure-source detection were

is now WARN (was FAIL). This prevents single narrow keywords from failing otherwise
